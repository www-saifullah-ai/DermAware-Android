## CORE FEATURES

### 1. PHOTO ANALYSIS (TFLite ML)
- Users can capture a photo using camera or select from gallery
- TFLite model analyzes the skin image on-device
- Returns top 3 possible matching conditions with confidence percentages
- Each result links to full condition information screen
- Camera guidance overlay helps user take good quality photos (good lighting tips, distance guide)
- Supports analyzing: face, arms, legs, scalp, hands, feet body areas
- IMPORTANT: Every result screen must show bold disclaimer: "This is for awareness only. Not a medical diagnosis. Consult a qualified dermatologist for proper evaluation."

### 2. SYMPTOM CHECKER
- Text-based questionnaire flow (no photo required)
- Step-by-step questions: body area affected, duration, appearance (color, texture, shape), associated symptoms (itching, pain, burning, spreading), age group, known allergies
- Smart branching logic based on answers
- Returns list of possible conditions matching symptoms with information cards
- Same disclaimer as photo analysis

### 3. CONDITION INFORMATION LIBRARY
- Comprehensive offline database of 50+ skin conditions covering:
  - Common infections: ringworm, scabies, impetigo, cellulitis, folliculitis, athlete's foot, nail fungus
  - Inflammatory diseases: eczema, psoriasis, seborrheic dermatitis, contact dermatitis, rosacea, acne
  - Dangerous conditions (awareness): melanoma warning signs (ABCDE rule), basal cell carcinoma signs, squamous cell carcinoma signs
  - Other common: vitiligo, hives, warts, molluscum, pityriasis versicolor, cold sores
- Each condition card includes:
  - Plain-language description
  - Common symptoms
  - Who is commonly affected
  - Home care guidance (what to do, what to avoid)
  - When to see a doctor (urgency: Low / Medium / High / Emergency)
  - Doctor referral advice
  - Educational images placeholder (use vector illustrations, no real patient photos)

### 4. HOME CARE GUIDANCE
- Each condition has a detailed home care section
- Includes: general hygiene tips, OTC medication categories (not brand names), lifestyle adjustments, what makes it worse
- Clearly labeled: "General Information Only — Not Medical Advice"
- Emergency warning signs that require immediate medical attention listed prominently

### 5. DOCTOR REFERRAL SYSTEM
- Urgency classification for every condition: 
  - 🟢 Low — monitor at home
  - 🟡 Medium — see a doctor within a week
  - 🔴 High — see a doctor soon (within 1-2 days)
  - 🚨 Emergency — seek immediate medical care
- "Find Help" section with guidance on types of doctors to visit (dermatologist, general practitioner, emergency room)
- Offline — no live doctor finder, but educational guidance on what kind of help to seek

### 6. ANALYSIS HISTORY
- All photo analyses and symptom checker results saved locally in Room database
- History screen with date, body area, top result
- Tap any history item to view full result again
- Delete individual records or clear all history
- Multiple user profiles supported (useful for families)

### 7. USER PROFILES
- Create profiles for family members (name, age group, skin type)
- Each profile has its own history
- No account required — all local

### 8. SKIN HEALTH EDUCATION
- Dedicated education section with articles on:
  - How to do a monthly skin self-check
  - Understanding the ABCDE rule for moles
  - Sun protection basics (UV awareness)
  - Skin hygiene for different climates
  - Skin care for children
- All content stored offline

### 9. SETTINGS
- Language selector (30+ languages)
- Dark mode / Light mode / System default
- Notification: monthly skin self-check reminder
- Profile management
- Clear history
- About / Disclaimer / Privacy Policy

## TFLITE ML IMPLEMENTATION

Since a pre-trained dermatology TFLite model needs to be sourced or trained separately, implement the ML pipeline as follows:

- Create a placeholder TFLite model integration using MobileNetV2 architecture as base
- Build complete MLManager class that handles: model loading from assets, image preprocessing (resize to 224x224, normalize), inference, and result parsing
- Structure the code so the actual .tflite model file can be dropped into assets/ml/ folder and it works immediately
- Use labels file (labels.txt) in assets/ml/ for condition mapping
- Implement confidence threshold: only show results above 15% confidence
- Show top 3 results ranked by confidence
- Handle model loading errors gracefully with user-friendly message
- Add model version tracking for future updates
